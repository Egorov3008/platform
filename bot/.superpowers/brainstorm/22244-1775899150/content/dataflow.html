Ключевые потоки данных

Как работают два главных сценария: вход через Telegram и покупка ключа

🔐 Поток: Вход через Telegram (Magic Link)
1
Браузер → POST /auth/telegram/request (tg_id или username)
2
Бэкенд генерирует one-time token → сохраняет в БД (TTL: 10 мин)
3
Бэкенд → Telegram Bot API → шлёт юзеру: https://web.app/auth?token=xxx
4
Юзер кликает ссылку → браузер: POST /auth/telegram/verify {token}
✓
Бэкенд возвращает access_token + refresh_token → пользователь авторизован
💳 Поток: Покупка ключа
1
Браузер → POST /payments/create {tariff_id}
2
Бэкенд создаёт платёж в YooKassa → получает payment_url → сохраняет в БД pending
3
Юзер платит на странице YooKassa
4
YooKassa → POST /payments/webhook (статус succeeded)
5
Бэкенд → 3x-UI API: создаёт клиента → получает ключ → сохраняет в БД
✓
Ключ появляется в личном кабинете пользователя
