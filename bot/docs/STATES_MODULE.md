# Модуль состояний (states)

Этот модуль содержит определения всех состояний FSM (Finite State Machine) для телеграм-бота, организованных в группы состояний.

## Общая структура

Каждая группа состояний наследуется от `StatesGroup` из aiogram и представляет собой конечный автомат для определенного сценария взаимодействия с пользователем.

Также в модуле определен класс `StateKey` - вспомогательный класс для работы с ключами состояний в реестре.

## Группы состояний

### MainMenu
Основное меню бота:
- `welcome` - Приветственное состояние
- `main` - Основное меню
- `min_main` - Минимальное основное меню

### GiftStates
Состояния для работы с подарками:
- `main` - Окно с ссылкой на подарок
- `activate_gift` - Подтверждение активации подарка
- `success` - Успешная активация
- `error` - Ошибка при активации
- `not_found` - Подарок не найден
- `already_used` - Подарок уже использован

### KeysInit
Состояния для управления ключами:
- `view_tariff` - Просмотр тарифа
- `confirmation_delete_key` - Подтверждение удаления ключа
- `create_key` - Создание ключа
- `create_trial` - Создание пробного ключа
- `create_gift_key` - Создание подарочного ключа
- `payment_form` - Форма оплаты
- `edition` - Редактирование
- `renewal` - Продление
- `list` - Список ключей
- `key` - Конкретный ключ
- `setting_tariff` - Настройка тарифа

### Instruction
Состояния для выбора инструкции по установке:
- `choosing_device` - Выбор устройства
- `android` - Инструкция для Android
- `iphone` - Инструкция для iPhone
- `windows` - Инструкция для Windows
- `linux` - Инструкция для Linux

### ReferralSistem
Состояния для реферальной системы:
- `main` - Основное меню реферальной системы
- `generate_form` - Форма генерации реферальной ссылки

### Register
Состояния для регистрации:
- `welcome` - Приветственное состояние регистрации
- `sending_registration` - Отправка регистрации
- `sender` - Отправитель

### UsageRules
Состояния для просмотра правил использования (по страницам):
- `main` - Основное состояние
- `page1` до `page9` - Постраничный просмотр правил

### Tariff
Состояния для работы с тарифами:
- `preview` - Предварительный просмотр тарифа

### PaymentState
Состояния для процесса оплаты:
- `view_tariff` - Просмотр тарифа
- `form_pay` - Форма оплаты
- `result_pay` - Результат оплаты
- `setting_pay` - Настройка оплаты

### AdminManager
Состояния для административной панели:
- `main` - Основное меню администратора
- `static_user` - Статистика пользователя
- `discharge_pay` - Списание средств
- `search` - Поиск
- `confirmation_deletion_keys` - Подтверждение удаления ключей
- `input_url` - Ввод URL

### AdminUserRegistration
Состояния для регистрации пользователя администратором:
- `user_registration_form` - Форма регистрации пользователя
- `choosing_server` - Выбор сервера

### AdminSearchManagementSG
Состояния для поиска пользователей администратором:
- `main` - Основное меню поиска
- `search_tg_id` - Поиск по Telegram ID
- `search_username` - Поиск по username
- `search_email` - Поиск по email

### AdminMassMailing
Состояния для массовой рассылки:
- `receiving_message` - Получение сообщения для рассылки
- `confirmation` - Подтверждение отправки

### AdminUserManagement
Состояния для управления пользователем администратором:
- `profile_user` - Профиль пользователя

### AdminKeyManagementSG
Состояния для управления ключами администратором:
- `key_list` - Список всех ключей
- `process_key_edit` - Процесс редактирования ключа
- `confirmation_delete_key` - Подтверждение удаления ключа
- `calendar` - Календарь
- `confirmation_select_date` - Подтверждение выбора даты
- `view_tariff` - Просмотр тарифа
- `confirmation_select_tariff` - Подтверждение выбора тарифа

## Использование

Группы состояний используются в хендлерах для управления потоком диалога с пользователем. Каждое состояние соответствует определенному этапу взаимодействия.

Пример использования:

```python
from states import MainMenu

@router.message(MainMenu.main)
async def main_menu_handler(message: Message):
    # Обработка сообщений в основном меню
    pass
```

## StateKey

Вспомогательный класс для работы с ключами состояний:

```python
class StateKey:
    def __init__(self, states_group: str, state: str):
        self.states_group = states_group
        self.state = state
    
    @property
    def full(self) -> str:
        """Возвращает полное имя состояния в формате Group:state"""
        return f"{self.states_group}:{self.state}"
```

Этот класс используется для создания уникальных ключей состояний, которые могут быть использованы в реестрах и других структурах данных.
