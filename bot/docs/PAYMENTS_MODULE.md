# Модуль payments

Модуль `payments` предназначен для работы с платежной системой ЮKassa, включая создание платежей, обработку вебхуков и подтверждение оплат.

## Структура модуля

- `__init__.py` - инициализация модуля
- `pay_config.py` - конфигурация и сервис для работы с ЮKassa
- `pyments_webhook.py` - обработка вебхуков от ЮKassa

## Конфигурация

Модуль использует следующие переменные из конфигурации:

- `YOOKASSA_SHOP_ID` - ID магазина в ЮKassa
- `YOOKASSA_SECRET_KEY` - секретный ключ для API ЮKassa
- `URL_BOT` - URL для возврата после оплаты
- `WEBHOOK_PATH` - путь для приема вебхуков

## Сервис YooKassService

Класс `YooKassService` предоставляет методы для работы с платежами:

### `create_payment_form(price: float, description: str) -> Dict[str, str]`

Создает новый платеж.

**Параметры:**
- `price` - сумма платежа в рублях
- `description` - описание платежа

**Возвращает:**
Словарь с:
- `payment_id` - идентификатор платежа
- `confirmation_url` - URL для подтверждения оплаты

### `get_waiting_for_capture(payment_id: str) -> Optional[str]`

Проверяет, ожидает ли платеж подтверждения.

**Параметры:**
- `payment_id` - идентификатор платежа

**Возвращает:**
- `waiting_for_capture` если платеж ожидает подтверждения
- `None` в противном случае

## Обработка вебхуков

Модуль `pyments_webhook.py` содержит сервис для обработки вебхуков от ЮKassa.

### Поддерживаемые события:

- `PAYMENT_SUCCEEDED` - успешная оплата
- `PAYMENT_WAITING_FOR_CAPTURE` - платеж ожидает подтверждения
- `PAYMENT_CANCELED` - платеж отменен
- `REFUND_SUCCEEDED` - успешный возврат средств

### Обработка успешного платежа

Когда приходит уведомление о успешной оплате, сервис:
1. Логирует событие
2. Передает управление `PaymentProcessor` через `PaymentRouter`

### Обработка платежа в ожидании подтверждения

Когда приходит уведомление о платеже, ожидающем подтверждения, сервис автоматически подтверждает платеж.

## Безопасность

Сервис проверяет IP-адрес отправителя вебхука с помощью `SecurityHelper` от ЮKassa, чтобы убедиться, что запрос пришел от доверенного источника.

## Инициализация

Для инициализации сервиса вебхуков используется асинхронная функция `init_webhook_service()`, которая:
- Создает контейнер зависимостей
- Регистрирует маршрут для вебхука
- Настраивает обработку запросов по указанному пути (`WEBHOOK_PATH`)
