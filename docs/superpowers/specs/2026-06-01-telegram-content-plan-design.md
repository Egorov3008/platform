# Дизайн: Контент-план Telegram-канала для VPN-бота

**Дата:** 2026-06-01
**Статус:** Утверждён пользователем, ожидает финального ревью
**Продукт:** VPN-платформа (`/home/admin/platform`) — FastAPI backend, Telegram-бот, веб-интерфейс, 3x-UI, YooKassa

## Контекст

Telegram-канал создаётся для продвижения бота VPN-сервиса. Бот — основной UI продукта, через который пользователи покупают/продлевают ключи, управляют подпиской. Цель канала — привлечение пользователей через ценный privacy-контент с минимальной прямой рекламой.

## Цели и ограничения

| Параметр | Решение |
|---|---|
| Тематика | Продукт/бизнес (VPN-бот), но контент ведётся как privacy-издание |
| Аудитория | Privacy/security: безопасники, журналисты, IT-ниндзя, ценители анонимности |
| Цель | Вирусный рост через пересылаемый ценный контент |
| Частота | 7 постов в неделю (ежедневно) |
| Формат | Короткие посты + визуал (без лонгридов) |
| Связь с ботом | Мягкая: бот упоминается в закрепе и профиле канала, в теле постов — только если тема прямо об этом |
| Подход | «Рубрикатор + слоты» (утверждён пользователем) |

## Архитектура контент-плана

Контент-план = 4 файла-артефакта в Obsidian (`second-brain/03-Projects/tg-channel-vpn/`), с разным ритмом обновления:

| Артефакт | Файл | Назначение | Обновляется |
|---|---|---|---|
| Расписание рубрик | `rubrics.md` | Что публикуется в какой день недели | Раз в 1-3 месяца |
| Сетка на неделю | `week-YYYY-MM-DD.md` | Конкретные темы на 7 дней | Каждое воскресенье |
| Банк тем | `ideas-bank.md` | Неиспользованные идеи (50+ на старте) | Пополняется постоянно |
| Шаблоны | `templates.md` | Структура каждого типа поста | По мере появления новых рубрик |
| Визуалы | `assets/` | Скриншоты, мемы, инфографики | По мере надобности |

Преимущества: разные ритмы обновления, возможность делегировать отдельный файл, Obsidian-совместимость (связи с `04-Permanent/`).

## 7 рубрик по дням недели

| День | Рубрика | Цель | Формат | Пример темы |
|---|---|---|---|---|
| ПН | 🔍 «Угроза недели» | Вирусность: страх → пересыл | Короткий разбор 1 угрозы (DPI, утечка, блок) | «Почему провайдерский VPN перестал работать в Турции — и при чём QUIC» |
| ВТ | 🛠 «Гайд на 60 секунд» | Сохранения: практическая польза | Пошаговая инструкция в 5-7 шагов со скриншотами | «60 секунд: настройка WireGuard на роутере Keenetic» |
| СР | 🧪 «Разбор мифа» | Экспертность: дискредитация маркетинга | Берём популярное утверждение → разбираем по фактам | «Миф: "VPN снижает скорость в 2 раза". Проверяем» |
| ЧТ | 📰 «Сводка privacy-новостей» | Реактивность: быть в теме | 3-5 новостей недели + наш комментарий | «Дайджест: DPI, утечки, новые протоколы» |
| ПТ | 💬 «Вопрос-ответ» | Вовлечение: community | Отвечаем на вопрос подписчиков | «Про TOR, фингерпринтинг и надёжность ключей» |
| СБ | 🎨 «Мем / визуал» | Вирусность: расшариваемость | 1 мем или 1 инфографика | «Когда сказали "мне VPN не нужен"» |
| ВС | 🪞 «За кулисами» | Доверие: humanize | Личное от автора/команды, честно про продукт | «Что мы починили за неделю в боте: 3 бага, 1 фича» |

**Принципы:**
- Ни одна рубрика не продаёт напрямую. Связь с ботом — через закреп и профиль.
- Повторяемый визуальный хедер → узнаваемость при пересылке.
- ПН + СБ = вирус (страх + юмор) → главный источник роста.
- ВТ + ЧТ = утилита (гайды + новости) → главный источник сохранений.
- СР + ПТ = вовлечение (мнение + диалог) → главный источник комментов.
- ВС = бэкстейдж → главный источник лояльности.

## Банк тем

`ideas-bank.md` — структурированный список, сгруппированный по рубрикам. Минимум 8-10 идей на рубрику = 56+ идей на старте. Формат — Obsidian-чеклисты: `- [ ] тема`. Использованные — `- [x]`. Это позволяет видеть, какие рубрики проседают.

## Шаблоны постов

`templates.md` — 7 шаблонов (по одному на рубрику). Каждый шаблон — скелет текста, который заполняется под конкретную тему. Шаблон гарантирует структуру (важно для пересылки), короткий текст, практический выход.

Пример для «Угрозы недели»:
```
🔍 УГРОЗА НЕДЕЛИ

<1 строка: что случилось / что обнаружено>

Почему это важно для тебя:
• <факт 1>
• <факт 2>
• <факт 3>

Что делать прямо сейчас:
→ <действие 1>
→ <действие 2>

<опционально: ссылка на полный разбор>
```

## Еженедельный процесс

**Кто делает:** автор канала лично на старте. Делегирование — после стабилизации.

**Воскресенье, 30-40 минут — «Планирование недели»:**
1. Открыть `rubrics.md` → какие рубрики на этой неделе
2. Из `ideas-bank.md` выбрать 7 тем (по 1 на рубрику), отметить чекбоксы
3. Если темы нет — добавить в банк (источники: RSS по privacy, утечки, новости регулирования)
4. Заполнить `week-YYYY-MM-DD.md` — 7 строк: день, рубрика, тема, ключевой тезис
5. Заготовить визуалы (скриншоты / мемы) в `assets/`

**Каждый день, 15-20 минут — «Сборка поста»:**
1. Открыть шаблон рубрики из `templates.md`
2. Заполнить по теме из `week-...md`
3. Прогнать чек-лист:
   - [ ] Иконка рубрики в начале
   - [ ] 1-2 эмодзи максимум
   - [ ] Подзаголовки через пустую строку (Telegram-рендеринг)
   - [ ] Нет «купи-купи», только факты/польза
   - [ ] Визуал прикреплён (если положено рубрикой)
   - [ ] Время публикации: 10:00 или 19:00 (пиковое для RU-аудитории)
4. Опубликовать
5. Собрать реакции/комменты → идеи для ВТ/ПТ

**Суббота, 10 минут — «Ретро недели»:**
- Сколько пересылок? (реакция 🔄)
- Какая рубрика зашла лучше?
- Добавить в банк тем идеи из комментов
- Подвинуть темы на следующую неделю при необходимости

**Принцип:** не пытаться публиковать идеально. Privacy-каналы растут, когда видно живого автора, а не корпоративный SMM.

## Метрики успеха

| Метрика | Как мерить | Цель на 1 мес | Цель на 3 мес |
|---|---|---|---|
| Прирост подписчиков | TGStat / встроенная статистика | +200-400 без рекламы | +1500-3000 |
| ERR (вовлечённость) | Просмотры ÷ подписчики на пост | 25-35% | 30-40% стабильно |
| Пересылки | Реакция 🔄 / статистика «поделились» | 10% постов с ≥5 пересылками | 30% постов с ≥10 |
| Сохранения | Реакция ⭐ / «сохранили» | 1 пост в неделю с ≥10 | 3+ поста в неделю |
| Переходы в бот | UTM в закрепе канала | Отслеживаем тренд | Рост коррелирует с ростом канала |

**Реалистичные сроки:**
- Неделя 1-2: канал «молчит» (20-100 подп.). Нормально — алгоритмы TG не любят пустые каналы. Главное — публиковать стабильно.
- Неделя 3-4: первые 2-3 виральных поста, ERR стабилизируется.
- Месяц 2: при стабильном контенте — органический рост 50-100/день.
- Месяц 3: можно пробовать платное продвижение (посевы в тематических чатах) с готовой базой.

**Красные флаги — когда менять стратегию:**
- ERR < 15% при 500+ подписчиках → контент не попадает в аудиторию, тестируем другие темы
- 0 пересылок за 2 недели подряд → посты слишком «корпоративные», нужны более эмоциональные рубрики
- Рост < 50 за месяц при стабильных публикациях → нужны коллабы/посевы

## Out of scope (в этом спеке НЕ делаем)

- Стратегия платного продвижения (только обозначено как опция на месяц 3)
- Дизайн визуалов (мемы, инфографики) — это content production, отдельный проект
- Интеграция с ботом (deep linking, автоматические приветствия) — можно добавить позже
- Коллаборации с другими каналами — это рост-тактика, а не контент-план
- Делегирование контент-мейкеру — после стабилизации процесса
